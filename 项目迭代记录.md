### 3.1 背景

在本次迭代前，项目已经具备基础博客能力，包括：

- 模块首页

- Markdown 渲染

#### 3.3.8 修复路由切换后正文空白问题

问题表现：

- 首次进入页面时正文正常
- 点击头部导航切换页面后，正文区域变空白
- 刷新后页面又恢复正常

定位结果：

- 路由视图被直接放在 `<Transition mode="out-in">` 中
- 多个页面组件返回的是 fragment，而不是单一元素根节点
- `<Transition>` 无法直接对 fragment 做过渡
- 切换时旧页面先卸载，新页面未能被正常承接，因此出现正文空白

修复方案：

- 为路由页面切换增加基于 `route.fullPath` 的唯一 key
- 在路由组件外层包裹真实的 DOM 容器，确保 `<Transition>` 处理的是单元素根节点

### 3.5 已知情况

当前仍有一些情况需要注意：

- 仍有大量旧文章尚未补充 Front Matter，构建时会按兼容规则降级处理
- 内容校验脚本会给出部分警告，但不影响当前站点运行
- 控制台中的 Vue 开发版提示、Chrome AI 提示、浏览器扩展 `runtime.lastError` 噪音，对站点功能本身没有实质影响
- 当前项目仍是 hash 路由 SPA，虽然已补充客户端 SEO 与 RSS/Sitemap，但单篇文章分享预览能力仍受静态 SPA 结构限制

### 3.6 后续建议

- 逐步为旧文章补齐 Front Matter，减少构建时的降级逻辑
- 增加生产环境资源版本切换，发布时改用 Vue 与 Vue Router 的生产版资源
- 补充 `favicon.ico`
- 如果后续继续强调 SEO 和分享预览，可以考虑引入预渲染或静态 HTML 产物输出
- 如果“设计模式”模块中的架构与工程实践文章继续增多，建议将附录进一步细分，避免分类语义持续扩散
