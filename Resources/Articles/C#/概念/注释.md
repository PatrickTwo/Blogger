## 1. 基本结构
```csharp
/// <summary>
/// 简要的功能描述
/// </summary>
/// <param name="parameterName">参数说明</param>
/// <returns>返回值说明</returns>
public void MethodName() { }
```

## 2. 常用标签详解

### `<summary>` - 核心功能描述
```csharp
/// <summary>
/// 将布尔值转换为可见性状态
/// </summary>
```

### `<remarks>` - 详细说明和原理
```csharp
/// <remarks>
/// 此转换器接收布尔值并返回对应的可见性状态。<br/>
/// 如果值为 true，返回 Visibility.Visible；<br/>
/// 如果值为 false，返回 Visibility.Hidden。
/// </remarks>
```

### `<example>` - 使用示例
```csharp
/// <example>
/// 以下示例展示如何在XAML中使用此转换器：
/// <code>
/// <Window.Resources>
///     <local:BoolToVisibilityConverter x:Key="Converter"/>
/// </Window.Resources>
/// <Image Visibility="{Binding IsChecked, Converter={StaticResource Converter}}"/>
/// </code>
/// </example>
```

## 6. 注释编写原则

1. **避免废话**：不要描述明显的代码逻辑（如"这是一个循环"）
2. **解释为什么**：说明特殊逻辑的原因（如跳过某些元素的原因）
3. **保持简洁**：summary应该一句话概括功能
4. **提供示例**：复杂的用法应该提供使用示例
5. **类型安全**：使用`<see cref="">`引用具体类型，而不是纯文本

## 7. 完整示例
```csharp
/// <summary>
/// 将 <see cref="bool"/> 值转换为 <see cref="Visibility"/> 枚举
/// </summary>
/// <remarks>
/// 此转换器接收布尔值并返回对应的可见性状态。<br/>
/// - true: <see cref="Visibility.Visible"/><br/>
/// - false: <see cref="Visibility.Hidden"/>
/// </remarks>
/// <example>
/// <code>
/// <Image Visibility="{Binding IsChecked, Converter={StaticResource BoolToVisibilityConverter}}"/>
/// </code>
/// </example>
/// <param name="value">输入的布尔值</param>
/// <returns>对应的可见性状态</returns>
public Visibility Convert(bool value)
{
    return value ? Visibility.Visible : Visibility.Hidden;
}
```

这样的注释不仅提供基本信息，还能在IDE中提供丰富的智能提示和导航功能。
