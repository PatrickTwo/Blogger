# 抽象类

抽象类是一种特殊的类，它不能被实例化，主要用于作为其他类的基类。抽象类的主要目的是为其子类提供一个统一的接口或模板，确保子类实现某些特定的方法或功能。

### 抽象类的特点：
1. **不能实例化**：抽象类不能直接创建对象，只能通过继承它的子类来使用，通过父类的应用指向子类的实例来间接的实例化子类
2. **抽象类可以包含普通属性和抽象属性，普通函数和抽象函数。**
3. **可以包含抽象方法**：抽象类中**可以**包含抽象方法，这些方法只有声明而没有实现，必须由子类去实现。
4. **可以包含具体方法**：抽象类中也可以包含具体实现的方法，这些方法可以直接被子类继承和使用。
5. **用于定义规范**：抽象类通常用于定义一组规范或接口，确保子类遵循这些规范。

### 抽象类的使用场景：
- **定义通用行为**：当多个类有共同的行为或属性时，可以将这些共同的部分提取到抽象类中，避免代码重复。
- **强制子类实现特定方法**：通过定义抽象方法，强制子类必须实现这些方法，确保子类的行为符合预期。
- **提供默认实现**：抽象类可以提供一些默认的实现，子类可以选择性地覆盖这些方法。

### 示例：
```java
// 定义一个抽象类
abstract class Animal {
    // 抽象方法，子类必须实现
    abstract void makeSound();

    // 具体方法，子类可以直接使用
    void sleep() {
        System.out.println("Sleeping...");
    }
}

// 子类继承抽象类
class Dog extends Animal {
    // 实现抽象方法
    void makeSound() {
        System.out.println("Bark");
    }
}

class Cat extends Animal {
    // 实现抽象方法
    void makeSound() {
        System.out.println("Meow");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal dog = new Dog();
        dog.makeSound(); // 输出: Bark
        dog.sleep(); // 输出: Sleeping...

        Animal cat = new Cat();
        cat.makeSound(); // 输出: Meow
        cat.sleep(); // 输出: Sleeping...
    }
}
```

### 总结：
抽象类是一种强大的工具，用于定义类的结构和行为规范。通过抽象类，可以确保子类遵循一定的设计模式，提高代码的可维护性和扩展性。
