## 一、核心概念：内存分配的不同

最根本的区别在于它们在内存中存储和传递的方式不同。

*   **值类型**：变量直接包含其数据。**值类型的实例存储在栈（Stack）上**（除了作为类的字段时，会随对象一起在堆上）。
*   **引用类型**：变量只存储对数据的**引用（内存地址）**，而实际数据存储在**堆（Heap）** 上。

---

## 二、值类型

#### 1. 包含哪些类型？
C# 中所有的值类型都派生自 `System.ValueType`（而 `System.ValueType` 又继承自 `System.Object`）。

*   **简单类型**：
    *   **整数类型**：`sbyte`, `byte`, `short`, `ushort`, `int`, `uint`, `long`, `ulong`
    *   **浮点类型**：`float`, `double`, `decimal`
    *   **字符类型**：`char`
    *   **布尔类型**：`bool`
*   **枚举类型**：`enum E {...}`
*   **结构体类型**：`struct S {...}`
*   **可空值类型**：`Nullable<T>`，例如 `int?`

#### 2. 特点与行为

示例 1：赋值操作 - “创建副本”
```csharp
int a = 10;
int b = a; // 将 a 的值 (10) 复制一份，赋给 b

Console.WriteLine($"a = {a}, b = {b}"); // 输出：a = 10, b = 10

b = 20; // 修改 b 的值

Console.WriteLine($"a = {a}, b = {b}"); // 输出：a = 10, b = 20
// a 的值没有因为 b 的改变而改变，因为它们是两个独立的存储单元。
```

示例 2：作为方法参数传递 - “按值传递”
```csharp
void ChangeValue(int number)
{
    number = 100; // 这个修改只在方法内部有效，不会影响外部的 x
}

int x = 50;
ChangeValue(x);
Console.WriteLine(x); // 输出：50
```

## 三、引用类型

#### 1. 包含哪些类型？
所有不是值类型的类型都是引用类型。它们都直接或间接继承自 `System.Object`。

*   **类**：`class C {...}`
*   **接口**：`interface I {...}` （注意：接口本身是引用类型）
*   **数组**：例如 `int[]`, `string[]`
*   **委托**：`delegate ...`
*   **字符串**：`string` （虽然它是不可变的，但类型是引用类型）

#### 2. 特点与行为

示例 3：赋值操作 - “共享同一份数据”
```csharp
public class Person
{
    public string Name;
}

Person p1 = new Person(); // 在堆上创建 Person 对象，p1 存储该对象的地址
p1.Name = "Alice";

Person p2 = p1; // 将 p1 的地址复制给 p2，现在 p1 和 p2 指向堆上的同一个对象！

Console.WriteLine($"p1.Name = {p1.Name}, p2.Name = {p2.Name}"); // 输出：p1.Name = Alice, p2.Name = Alice

p2.Name = "Bob"; // 通过 p2 修改了堆上对象的属性

Console.WriteLine($"p1.Name = {p1.Name}, p2.Name = {p2.Name}"); // 输出：p1.Name = Bob, p2.Name = Bob
// p1.Name 也变成了 "Bob"，因为它们指向的是同一个对象。
```

示例 4：作为方法参数传递 - “按引用传递地址”
```csharp
void ChangeName(Person person)
{
    person.Name = "Charlie"; // 通过引用修改了堆上对象的属性
}

Person p3 = new Person();
p3.Name = "David";
ChangeName(p3);
Console.WriteLine(p3.Name); // 输出：Charlie
```

## 区别表

| 特性         | 值类型                                                | 引用类型                                      |
| :----------- | :---------------------------------------------------- | :-------------------------------------------- |
| **存储内容** | 实际数据                                              | 数据的引用（内存地址）                        |
| **内存位置** | 栈（或内联于包含它的对象所在的堆中）                  | 堆                                            |
| **内存管理** | 超出作用域自动释放（栈机制）                          | 由垃圾回收器（GC）管理                        |
| **默认值**   | 数值为 0，布尔为 `false`，结构为成员默认值            | `null`                                        |
| **赋值操作** | 创建数据的**副本**                                    | 复制**引用**，指向同一对象                    |
| **比较操作** | 比较**数据内容**是否相等（`==` 对于基本类型）         | 比较**引用地址**是否相同（除非重写 `Equals`） |
| **继承性**   | 隐式密封，不能继承自其他类型（除 `System.ValueType`） | 可以继承自其他类                              |

## 五、特殊情况

#### 1. `string` 的特殊性
`string` 是引用类型，但它是**不可变的**。任何看似修改字符串的操作（如 `ToUpper()`, `Substring()`），实际上都会返回一个**新的字符串对象**。

```csharp
string s1 = "hello";
string s2 = s1; // s2 和 s1 指向同一个 "hello" 字符串

s1 = "world"; // 这并非修改原字符串，而是让 s1 指向了一个全新的 "world" 字符串

Console.WriteLine(s1); // world
Console.WriteLine(s2); // hello （s2 依然指向原来的 "hello"）
```

#### 2. `ref` 和 `out` 关键字
默认情况下，方法的参数是**按值传递**的。对于值类型，传递的是副本；对于引用类型，传递的是引用的副本（但你仍然可以通过这个副本修改对象的内容）。

使用 `ref` 关键字可以强制实现**按引用传递**。这意味着：
*   对于**值类型**：传递的不是副本，而是原始变量的内存地址。方法内对参数的修改会直接影响原始变量。
*   对于**引用类型**：传递的是引用的引用（指针的指针）。你可以在方法内部让这个引用指向一个全新的对象，并且这个改变会影响外部的原始引用。

```csharp
// 值类型使用 ref
void Increment(ref int value)
{
    value++;
}
int num = 5;
Increment(ref num);
Console.WriteLine(num); // 输出：6

// 引用类型使用 ref
void ChangeReference(ref Person p)
{
    p = new Person { Name = "New Object" }; // 这会修改外部 pRef 的指向
}
Person pRef = new Person { Name = "Old Object" };
ChangeReference(ref pRef);
Console.WriteLine(pRef.Name); // 输出：New Object
```

#### 3. 参数传递的总结
*   **值传递（默认）**：形参是实参的副本。对形参的修改不影响实参。
*   **引用传递（`ref`）**：形参是实参的别名。对形参的任何操作就是对实参的操作。必须在调用和声明时使用 `ref`。
*   **输出传递（`out`）**：类似于 `ref`，但更侧重于从方法中**输出**值。必须在方法内部对其赋值。

### 六、如何选择：使用值类型还是引用类型？

*   **优先使用值类型的情况**：
    *   类型的实例较小（通常小于 16 字节）。
    *   实例是短暂的，不常在方法间传递。
    *   不需要多态行为（值类型不能被继承）。
    *   希望数据是独立的、不可变的副本（避免意外的副作用）。例如，`int`, `Point`, `Color` 等。

*   **优先使用引用类型的情况**：
    *   类型的实例较大，复制成本高。
    *   需要多态和继承。
    *   需要通过多个引用访问和修改同一个对象（共享状态）。例如，`List<T>`, `Stream`, `StringBuilder` 等。

理解值类型和引用类型的差异，能帮助你预测代码的执行结果，避免常见的陷阱（如意外修改共享数据），并做出更合理的数据结构设计决策。
