## 一、扩展方法基础概念

#### 什么是扩展方法
- **定义**：向现有类型添加新方法，无需修改原类型

####  声明语法
静态方法+关键字`this`
```csharp
// 静态类命名规范：被扩展类名 + Extensions
public static class UserExtensions
{
    // 静态方法 + this关键字
    public static bool IsValid(this User user)
    {
        return !string.IsNullOrEmpty(user.Name) && user.IsActive;
    }
}
```

## 二、核心特性与用法

#### 基本使用
```csharp
// 传统调用方式
UserExtensions.IsValid(user);

// 扩展方法调用方式（推荐）
user.IsValid();
```

#### 编译器标识
- 扩展方法在IDE中显示**小箭头图标**
- Visual Studio/Rider中标注"（扩展）"

#### 为基本类型扩展
```csharp
public static class IntExtensions
{
    public static bool IsEven(this int value) => value % 2 == 0;
    public static int Square(this int value) => value * value;
}

// 使用
int value = 10;
value.IsEven();    // true
value.Square();    // 100
```

## 三、重要注意事项

### 3.1 命名冲突规则
- 扩展方法与类成员方法同名时，**优先调用类成员方法**
- 强制调用扩展方法：通过静态类直接调用
```csharp
user.Intro();                    // 调用类成员方法
UserExtensions.Intro(user);     // 调用扩展方法
```

### 3.2 反射调用
```csharp
// 错误方式：在目标类上查找
typeof(User).GetMethod("Intro", BindingFlags.Instance | BindingFlags.Public);

// 正确方式：在静态扩展类上查找
typeof(UserExtensions).GetMethod("Intro", BindingFlags.Static | BindingFlags.Public);
method.Invoke(null, new object[] { user });  // 第一个参数传null
```

### 3.3 访问权限限制
- 扩展方法**无法访问**类的私有成员
- 位于类外部，受C#访问修饰符限制

## 四、高级应用场景

### 4.1 鸭子类型实现
```csharp
// 通过扩展方法让自定义类型支持await
public static class MyDelayExtensions
{
    public static TaskAwaiter GetAwaiter(this MyDelay delay)
    {
        return Task.Delay(1000).GetAwaiter();
    }
}

// 甚至为基础类型添加await支持
public static class TimeSpanExtensions
{
    public static TaskAwaiter GetAwaiter(this TimeSpan timeSpan)
    {
        return Task.Delay(timeSpan).GetAwaiter();
    }
}
```

### 4.2 集合表达式支持
```csharp
public static class MyCollectionExtensions
{
    public static void Add<T>(this MyCollection<T> collection, T item)
    {
        collection.Push(item);  // 调用内部方法
    }
}
// 使得自定义集合支持新的初始化语法：MyCollection<int> col = [1, 2, 3];
```

### 4.3 服务注册模式（ASP.NET Core风格）
```csharp
public static class ServiceCollectionExtensions
{
    public static IServiceCollection AddCustomServices(this IServiceCollection services)
    {
        services.AddSingleton<IMyLog, MyLog>();
        services.AddTransient<IMyCustomService, MyCustomService>();
        return services;  // 支持链式调用
    }
}

// 使用
services.AddCustomServices()
        .AddOtherServices();
```

## 五、封装最佳实践

### 5.1 命名空间隔离
```csharp
namespace MyCompany.Extensions
{
    public static class StringExtensions
    {
        public static string Reverse(this string str) => new string(str.Reverse().ToArray());
    }
}

// 按需引入：using MyCompany.Extensions;
```

### 5.2 避免类型污染
- 不要将扩展方法放在常用命名空间（如System）
- 根据使用场景合理组织命名空间结构
- 参考LINQ的设计：默认不引入，需要时using System.Linq

## 六、C# 14新特性：扩展成员

### 6.1 新语法格式
```csharp
public static class StringExtensions
{
    // 扩展属性（新语法）
    extension string Source
    {
        public bool HasContent => !string.IsNullOrEmpty(Source);
    }
    
    // 扩展静态方法（新语法）
    extension string
    {
        public static int WordCount(string sentence) => sentence.Split(' ').Length;
    }
}
```

### 6.2 优势对比
| 特性 | 传统扩展方法 | 新扩展成员 |
|------|-------------|------------|
| 实例方法 | ✅ | ✅ |
| 属性支持 | ❌ | ✅ |
| 静态方法 | ❌ | ✅ |
| 语法简洁性 | 一般 | 更好 |

### 6.3 底层实现
- 仍然是**语法糖**，编译为静态类方法
- 实例成员编译为：`StringExtensions.GetHasContent(s1)`
- 静态成员编译为：`StringExtensions.WordCount(s1)`

## 七、适用场景总结

1. **第三方库类型扩展**：无法修改源码的DTO、Entity类
2. **基础类型增强**：int、string等系统类型的工具方法
3. **架构封装**：依赖注入注册、服务配置
4. **语法糖实现**：让自定义类型支持新语法特性
5. **代码组织**：将相关功能逻辑分组封装

## 八、核心要点回顾

- 🎯 **本质**：静态类的静态方法，通过this关键字提供语法糖
- 🔧 **反射**：需要在静态扩展类上操作，而非目标类
- 🚫 **限制**：无法访问私有成员，优先级低于类原生方法
- 📦 **封装**：通过命名空间控制可见性，避免污染
- 🆕 **发展**：C# 14引入扩展成员，支持属性和静态方法扩展

通过合理使用扩展方法，可以显著提升代码的可读性和可维护性，同时保持对现有代码的零侵入性。
